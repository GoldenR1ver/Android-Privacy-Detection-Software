# Android-Privacy-Detection-Software

Android Privacy Detection Software. Used as part of the Nanjing University Computer Systems Experiment to detect Android privacy policies.



## 隐私政策分析依据

将《个人信息保护法》对隐私政策文件的内容和组织方式要求概括如下：

### 1. 核心告知内容：八大必备要素

根据《个保法》第十七条，一份合规的隐私政策必须包含以下信息，并且必须遵循**真实、准确、完整**的原则 ：

- **处理者身份**：个人信息处理者的名称或者姓名和联系方式。

- **处理目的**：处理个人信息的目的。目的必须明确、合理，不能笼统模糊（如“改善服务”）。

- **处理方式**：个人信息的处理方式（如收集、存储、使用、加工、传输、提供、公开、删除等）。

- **信息种类**：处理的个人信息种类。需要逐项列举，不能使用“等、例如”字样概括 。

- **保存期限**：个人信息的保存期限。法律要求保存期限应为实现处理目的所必要的最短时间 。

- **用户权利**：个人行使法定权利（查阅、复制、更正、删除、撤回同意等）的方式和程序 。

- **法律规定的其他事项**：法律、行政法规规定应当告知的其他事项。

### 2. 告知形式要求：清晰易懂、显著透明

除了内容，法律对告知的形式也做了强制性规定，旨在消除信息不对称]：

- **显著方式**：个人信息处理者应当以个人容易辨识且易于获取的方式让个人了解告知内容，不能将其隐藏在冗长的政策中，或以极小、难以辨识的字体呈现。

- **清晰易懂的语言**：必须使用普通人能理解的语言表述，避免使用晦涩、抽象或模糊的专业术语（如“增强安全性”“提升用户体验”等无法说明具体目的的词汇）。

- **便于查阅和保存**：如果通过制定个人信息处理规则的方式告知，该规则应当公开，并且便于查阅和保存。

### 3. 特殊场景下的“加强版”告知要求

当涉及敏感个人信息或特定处理环节时，法律规定了额外的告知义务，且通常需要取得用户的**单独同意**：

#### **（1）处理敏感个人信息**

敏感个人信息包括生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等信息，以及不满十四周岁未成年人的个人信息。

- **额外告知内容**：除核心告知事项外，还必须向个人告知**处理敏感个人信息的必要性**以及对个人权益的**影响**。

- **同意形式**：必须取得个人的**单独同意**，不得与一般信息捆绑授权。涉及不满十四周岁未成年人信息的，还需取得监护人同意，并制定专门的个人信息处理规则。

#### **（2）向第三方提供个人信息**

- **额外告知内容**：需向个人告知**接收方的名称或者姓名、联系方式、处理目的、处理方式和个人信息的种类** 。

- **同意形式**：取得个人的**单独同意**。

#### **（3）跨境提供个人信息**

- **额外告知内容**：需向个人告知**境外接收方的名称或者姓名、联系方式、处理目的、处理方式、个人信息的种类**以及个人向境外接收方行使法定权利的**方式和程序**。

- **同意形式**：取得个人的**单独同意**。

#### **（4）自动化决策**

- **透明度要求**：利用个人信息进行自动化决策（如“大数据杀熟”、个性化推送），应当保证决策的**透明度和结果公平、公正**。

- **拒绝权告知**：通过自动化决策方式进行信息推送、商业营销的，应当同时提供**不针对其个人特征的选项**，或者向个人提供便捷的**拒绝方式**。

#### **（5）个人信息转移**

- **因合并、分立等原因**：个人信息处理者因合并、分立、解散、被宣告破产等原因需要转移个人信息的，应当向个人告知**接收方的名称或者姓名和联系方式**。

### 4. 例外情况与变更告知

- **免于告知**：法律、行政法规规定应当保密，或者不需要告知的情形，可以不告知。

- **紧急情况**：紧急情况下为保护自然人的生命健康和财产安全无法及时告知的，应当在紧急情况消除后及时告知。

- **变更告知**：如果处理目的、处理方式、信息种类等核心事项发生变更，应当将变更部分告知个人，并重新取得同意。

### 总结

《个人信息保护法》对隐私政策的要求可以概括为：**在事前，以显著、清晰的方式，全面、真实地告知处理者信息、处理目的、信息种类、保存期限、用户权利等核心内容；在此基础上，针对敏感信息、第三方共享、跨境传输等高风险场景，增加必要性及影响的告知，并获取用户的单独同意。** 任何信息的变更都需要及时重新告知并获取同意。
